User journey
8 steps, 2 human decisions
The system does the heavy lifting. Users decide sources and artifacts — nothing else.
1

Persona selection

Content Creator or Product Manager. Routes to the right agent crew.

2

Topic input

Free text — broad or specific. The system handles both.

3

Context enrichment optional

Upload PDFs, images, docs · paste a social handle · drop a website URL. Multimodal. Enriches the brief without replacing source discovery.

4

Clarification loop lightweight agent

Targeted questions one at a time. Skips what context already answered. Stops when brief is clear.

5

Source curation human decision 1

System surfaces vetted candidates with confidence parameters (recency · domain authority · relevance · cross-validation). User sets threshold and selects sources.

6

Artifact selection human decision 2

Pick what you want out. Only selected artifacts are generated — no waste, lower cost.

7

Agent crew runs

LangGraph orchestration. Retrieval → Analysis → Output. Only against selected sources.

8

Structured output delivered

Persona-specific artifacts with citations. Context-aware — reflects your voice, audience, gaps.

Design decisions
Every choice has a reason
The decisions that shaped the architecture — and why we made them.
Decision
Separate crews per persona with shared infrastructure
Why not the alternative
Shared crew with persona-specific output only — retrieval and analysis are fundamentally different per persona. A CC needs social sources and narrative hooks; a PM needs market data and evidence validation. Generic retrieval produces generic output.
What it enables
Each crew is perfectly tuned. Teams work in parallel without conflicts. Adding a persona = 3 agents + subclass. Infrastructure maintained once.
Decision
User selects sources + artifacts — system never decides for them
Why not the alternative
Auto-selecting sources and generating all artifacts — hides confidence scoring, produces waste, and removes user trust. In research, transparency matters more than convenience.
What it enables
User trusts the output because they chose the inputs. Only selected agents run — lower cost, faster results. Confidence parameters are a differentiating feature, not a detail.
Decision
OpenAI-compatible interface with model routing by agent role
Why not the alternative
Hardcoding one model — locks the system to one provider, one cost tier, one capability ceiling. Claude, GPT-4o, and open-source models all expose the same interface today.
What it enables
Fast/cheap model for intake, powerful model for analysis. Model swap = config change, not code change. User can choose their own model tier.
Decision
Tavily for discovery, BeautifulSoup only on selected URLs
Why not the alternative
Scraping everything with BeautifulSoup — slow, fragile, and wasteful. Scraping only after user selection means we deep-scrape exactly the sources that matter.
What it enables
Fast source discovery via Tavily API. Deep content only from chosen sources. Respects user's source selection decision. Lower API cost overall.
Decision
LangFuse over LangSmith for observability
Why not the alternative
LangSmith is proprietary and usage-capped on free tier. For a production system we intend to keep running, open-source observability with a self-host path is the right long-term call.
What it enables
Cloud free tier now, self-hosted on BharatDesha infra later — one config change, no code changes. Full agent trace visibility in the Loom demo.

The architectural guarantee
Every design decision in Phase 1 was made with Phases 2 and 3 in mind. The BaseCrew contract, the OpenAI-compatible interface, the env separation, the shared infrastructure layer — none of these are over-engineering. They are the minimum viable architecture for a system designed to grow.
