User journey — Multi-Agent Deep Researcher

The guided intake flow before agents start working

1
Persona selection
User picks their role — Content Creator or Product Manager. Sets the agent crew and output options available.
2
Topic input
Free text — broad or specific. "Sustainable travel in Rajasthan" or "Gen Z buying behavior in India."
3
Clarification loop lightweight agent
System asks targeted questions one at a time — audience, tone, angle, depth — until the brief is clear enough to proceed. User can skip at any point.
4
Source curation pause point
System surfaces a curated list of vetted, relevant sources. User selects which ones to include. Agents only search within selected sources — no hallucinated citations.
5
Artifact selection
User picks what they want out — content brief, social draft, captions, hashtags, PM brief, competitive summary, etc. Only selected artifacts are generated.
6
Agent crew runs LangGraph orchestration
Retrieval → Analysis → Synthesis → Report Builder agents run in sequence, only against selected sources, only producing selected artifacts.
7
Structured output delivered
Clean, persona-specific artifacts with source citations. Downloadable. Ready to use.
Key architectural insight
Steps 1–5 are cheap (one lightweight intake agent). The heavy retrieval + synthesis agents only fire at step 6, against a well-scoped brief. This minimises cost and maximises output quality.
