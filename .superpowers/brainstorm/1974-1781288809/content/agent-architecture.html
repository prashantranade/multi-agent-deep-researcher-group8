Agent crew architecture — A vs B

Click your preferred approach

A

Shared core, persona output

One agent crew handles intake, retrieval, and analysis. At the final stage, a persona-specific output agent shapes the results for Content Creator vs Product Manager.

[ Intake Agent ] → [ Retrieval Agent ] → [ Analysis Agent ]
↓
[ Content Creator Output Agent ]
[ Product Manager Output Agent ]

Pros

  • Simpler to build — one retrieval + analysis pipeline
  • Less duplication — core agents maintained once
  • Adding a new persona = adding one output agent only
  • Easier to test — core pipeline is persona-agnostic
  • Lower token cost — shared retrieval runs once

Cons

  • Analysis is generic — not tuned to what each persona actually needs
  • A PM needs competitive framing; a creator needs narrative hooks — shared analysis misses this
  • Output agent carries too much responsibility
B

Fully separate crews per persona

Each persona has its own end-to-end agent crew — different retrieval strategy, different analysis lens, different output format. No shared agents.

Content Creator: [ CC Retrieval ] → [ CC Analysis ] → [ CC Output ]
Product Manager: [ PM Retrieval ] → [ PM Analysis ] → [ PM Output ]

Pros

  • Each crew is perfectly tuned — retrieval sources, analysis depth, and output format all match the persona
  • CC crew searches social/cultural sources; PM crew searches market/industry sources
  • Teams can build in parallel — no coordination on shared agents
  • Failures are isolated — one crew breaking doesn't affect the other

Cons

  • More to build — double the agents
  • Duplication in retrieval logic that's actually the same
  • Adding a new persona = building an entire new crew
  • Higher risk of inconsistent quality across crews
The real question
Is the difference between personas mainly in how results are presented (favours A), or in what gets retrieved and how it's analysed (favours B)? A content creator researching "Gen Z travel" needs different sources and a different analytical lens than a PM researching the same topic.
